## Описание

### Актуальность
В наше время приходиться читать огроминое количество научных трудов. Наш продукт структурирует всю информацию с выданного ему текста, возвращая файл написанный на языке **Markdown**.

### Цель
Мы поставили перед собой цель написать скрипт, который будет обращаться к [**API Gigachat**](https://developers.sber.ru/portal/products/gigachat-api) для обработки данных в формат читаемый приложением [**Obsidian**](https://obsidian.md/).

### Планы
Мы предлагаем сделать расширение для приложения **Obsidian**, который будет структурировать документации или статьи выбираемые пользователем в **Markdown**. Планируется сделать произвольный формат данных на вход: **txt**, **docx**, **html**, **pdf**. Также хотелось бы реализовать добавление новых конспектов с ссылками на уже существующие.

### Функциональность
Проект включает в себя модуль для записи структурированных данных в формате Markdown с YAML-фронт-маттером. Поддерживаются различные типы данных: определения, теоремы, формулы, примеры, законы, принципы и другие. Формулы автоматически оборачиваются в LaTeX-разметку.

Пример использования:
```python
from src.in_out.markdown_writer import save_to_markdown
from src.api.time_from_api import get_current_time

# Получаем структурированные данные от GigaChat
structured_data = {
    'definitions': ['Определение 1', 'Определение 2'],
    'theorems': ['Теорема 1'],
    'formulas': ['E = mc^2'],
    'examples': ['Пример 1']
}

# Сохраняем в Markdown файл
time = get_current_time("Europe/Moscow")
save_to_markdown(structured_data, "output.md", time, ['Математика'], ['алгебра'])
```

### Ход работы
На текущий момент написано подключение к **API Gigachat**. Написаны **prompt** для нескольких агентов **(математика, физика, машинное обучение и искусственный интеллект, электротехника, материаловеденье, механика)**. Реализованн тест, в котором агент-математик сттруктурирует входные данные.
